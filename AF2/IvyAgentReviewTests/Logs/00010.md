## Reflection
- The `Html` component rendering issue is a recurring risk for apps that use raw HTML with CSS variables. Future test generation should flag `new Html(...)` usage as a potential rendering issue.
- `keyboard.type()` is more reliable than `fill()` for Ivy TextInput when the app checks input length to trigger side effects (like starting a timer).
- SelectInput options with shared prefixes always need `{ exact: true }` — this should be a default pattern.
