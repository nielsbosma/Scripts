# MakePlan - Session 00007

## Args
When I launch new Ivy apps I'm forced to allow it through the firewall. Can that be turned off?

## Outcome
Created plan **351** - Fix Windows Firewall Prompt on Ivy App Launch

Root cause: `Server.cs` line 509 uses `http://*:{port}` (all interfaces) instead of `http://localhost:{port}` (loopback only), triggering Windows Firewall prompts.

Fix: Change wildcard bind to localhost in `Ivy-Framework\src\Ivy\Server.cs`.
